# Этап 1. Сбор данных

## Цель

Собрать открытые данные о закупках юридических лиц группы Сбер за 2024–2025
годы, связать несколько источников, обогатить реквизиты и обезличить
персональные данные.

## Конфигурация

- `config/sber_organizations_seed.csv` — реестр юридических лиц и ИНН;
- `config/eis_search_queries.csv` — поисковые запросы для ЕИС;
- `config/source_probe_urls.csv` — адреса проверяемых площадок;
- `config/sources.json` — роль источника и причина его использования.

## Источники и их роль

- Сбербанк-АСТ и SberB2B дают основной корпоративный массив карточек;
- ЕИС даёт официальные сведения по процедурам 44-ФЗ и 223-ФЗ;
- ФНС ЕГРЮЛ подтверждает ИНН, КПП, ОГРН и юридическое наименование;
- остальные ЭТП проверяются для оценки покрытия и поиска дополнительных данных.

Сбербанк-АСТ выбран основным источником, потому что содержит наибольшее число
подтверждённых процедур группы Сбер. ЕИС используется для независимой проверки
и межисточникового обогащения.

## Скрипты

- `discover_egrul.py` — проверка организаций через ЕГРЮЛ;
- `collect_eis.py` — сбор ЕИС по организациям;
- `collect_eis_queries.py` — расширенный поиск кандидатов в ЕИС;
- `collect_sberbank_ast.py` — сбор Сбербанк-АСТ;
- `probe_sources.py` — проверка площадок;
- `build_stage1_dataset.py` — объединение выгрузок;
- `anonymize.py` — обезличивание текста;
- `anonymize_exports.py` — формирование обезличенных файлов;
- `audit_anonymization.py` — контроль остаточных персональных данных;
- `enrich_multisource.py` — связывание ЕИС и Сбербанк-АСТ;
- `report_stage1_coverage.py` — покрытие по годам и организациям;
- `summarize_stage1.py` — итоговая статистика.

## Связывание источников

Основной файл:

`data/processed/stage1/purchases_multisource_enriched.csv`

Связь выполняется по точному номеру закупки. Текстовое сходство само по себе не
используется для автоматического объединения. Найдено 3 точных пары
ЕИС ↔ Сбербанк-АСТ. Аудит связей хранится в `source_links.csv`.

## Обезличивание

Маскируются:

- ФИО и инициалы;
- телефоны и электронная почта;
- паспортные данные;
- СНИЛС;
- ИНН физических лиц.

ИНН и наименования юридических лиц, номера закупок и технические идентификаторы
сохраняются, поскольку нужны для связывания и аналитики.

## Запуск

```powershell
python stages\stage1\scripts\collect_eis.py
python stages\stage1\scripts\collect_eis_queries.py --pages 10
python stages\stage1\scripts\collect_sberbank_ast.py --query "Сбербанк"
python stages\stage1\scripts\probe_sources.py
python stages\stage1\scripts\build_stage1_dataset.py
python stages\stage1\scripts\anonymize_exports.py
python stages\stage1\scripts\audit_anonymization.py
python stages\stage1\scripts\enrich_multisource.py
python stages\stage1\scripts\summarize_stage1.py
```

## Результат

- 17 организаций;
- 1 458 подтверждённых закупок;
- 325 процедур за 2024 год;
- 1 133 процедуры за 2025 год;
- 3 точные межисточниковые связи;
- обогащение КПП и ОГРН для канонических записей;
- ноль находок высокого риска после аудита обезличивания.
