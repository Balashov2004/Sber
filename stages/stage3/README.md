# Этап 3. Аналитический модуль

## Данные

Модуль использует 1 458 канонических закупок и официальные ежедневные ряды
Банка России за 2024–2025 годы.

## Ключевое направление

Выбраны IT и телеком:

- 210 процедур;
- 14,40% выборки;
- 906,82 млн руб. указанной стоимости;
- 61 процедура в 2024 году;
- 149 процедур в 2025 году.

Направление выбрано из-за экономической значимости и возможной зависимости
импортного оборудования и лицензий от USD/RUB.

## Сравнение годов

- все закупки: 325 в 2024 году и 1 133 в 2025 году;
- IT и телеком: рост количества на 144,26%;
- сумма IT и телеком: рост на 254,88%;
- строительные закупки выросли по количеству, но не по сумме.

Рост интерпретируется осторожно, поскольку покрытие источника по годам
различается.

## Корреляции

Проверяются:

1. сумма IT-закупок и USD/RUB;
2. количество IT-закупок и USD/RUB;
3. сумма строительных закупок и ключевая ставка;
4. количество строительных закупок и ключевая ставка.

Для сумм применяется `log1p`. Используется коэффициент Пирсона и
перестановочный тест на 10 000 перестановок.

Результат:

- сумма IT и USD/RUB: значимой связи нет;
- количество IT и USD/RUB: умеренная отрицательная связь;
- строительство и ключевая ставка: значимой связи нет.

Корреляция не считается доказательством причинности.

## Аномалии

Формируются списки:

- экстремальных сумм;
- отменённых процедур;
- повторных публикаций после отмены;
- конфликтов статуса;
- двукратных и более скачков цены похожего предмета.

Сигнал является поводом для проверки, а не доказательством нарушения.

## Локальная Qwen

Qwen 3.5 9B запускается через Ollama:

1. Скачать Ollama: `https://ollama.com/download/windows`.
2. Установить приложение.
3. Выполнить:

```powershell
ollama pull qwen3.5:9b
ollama list
```

4. Запустить анализ:

```powershell
.\run_stage3_qwen.ps1 -Limit 25
```

Модель структурирует карточки в JSON и формирует черновик вывода. Ответы
проверяются отдельным скриптом:

```powershell
python stages\stage3\scripts\summarize_llm_results.py
```

LLM не используется как источник фактов об участниках, победителях,
единственном поставщике или нарушениях.

Каждый результат сохраняется сразу в `llm_results.jsonl`. Повторный запуск
продолжает очередь. Для запуска через кнопку используется
`run_stage3_qwen.cmd`.

## Запуск аналитики
